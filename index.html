I Encoded Categories by How They Behave, Not by What They Are

Target-based encoding for high-cardinality support-case metadata

View Repository Explore Method

Overview

This project contained multiple high-cardinality support-case metadata fields such as site, company, product, and user group.

One-hot encoding would explode dimensionality. Label encoding would introduce fake numeric order. So categories were encoded through their historical escalation behavior instead.

Architecture

The Core Idea

A category value is not useful only because it is different from the others.

In many applied problems, what matters is how that category historically behaves relative to the target.

Encoding can represent category behavior instead of category identity.

Selected code snippet

rates <- train[, .(
    not_esc = sum(!(REFERENCEID %in% ref_ids_esc)),
    esc = sum(REFERENCEID %in% ref_ids_esc)
), by = SITENAME]

rates[, site_enc := not_esc / (not_esc + esc) * 100]

Why this mattered

  • avoided thousands of sparse dummy variables
  • avoided false ordinal meaning from label encoding
  • let a single numeric column capture category behavior
  • worked naturally with tree-based models and high-cardinality metadata

Leakage awareness

Target-based encodings were computed on training data only, or handled with CatBoost-style ordered logic, to avoid leaking category outcomes back into the same rows.

Key Takeaway

I encoded categories by how they behave, not by what they are.

Built as a public project showcase for behavior-based encoding of high-cardinality categorical variables.
